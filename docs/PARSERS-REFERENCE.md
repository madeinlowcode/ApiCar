# Parsers Reference

## How Parsers Work

Each parser receives an ARIA snapshot of a page (text representation of the DOM)
and extracts structured data from it. The ARIA snapshot format is generated by
Playwright's `page.accessibility.snapshot()`.

All parsers inherit from `BaseParser` and implement the `parse(content, url)` method.

## Level 1: HomepageParser (`crawler/parsers/homepage.py`)

**Input**: catcar.info homepage
**Output**: Brand entries grouped by region

Extracts brands from 4 region groups:
- Europe (VW, Audi, BMW, Mercedes, etc.)
- Japan (Toyota, Honda, Nissan, etc.)
- Korea (Hyundai, Kia)
- USA (Ford, GM, etc.)

Each brand has: name, slug, region, catalog_path, catalog_url

## Level 2: BrandModelsParser (`crawler/parsers/brand_models.py`)

**Input**: Brand catalog page (table-based)
**Output**: Model entries with production info

Parses HTML tables with columns: Code, Model, Production Date, Codes
Used for VW-group brands and others with tabular model lists.

## Level 2 (alt): LinkModelsParser (`crawler/parsers/link_models.py`)

**Input**: Brand catalog page (link-based)
**Output**: Model entries from link lists

Used for BMW, Ford, and other brands that list models as links rather than tables.

## Level 2 (gateway): MarketSelectionParser (`crawler/parsers/market_selection.py`)

**Input**: Gateway page with market/brand tabs
**Output**: List of URLs to follow (re-enqueued at L2)

Some brands show a market selection page before models. The engine detects this
via h1 heading and follows the links.

## Level 3: ModelYearsParser (`crawler/parsers/model_years.py`)

**Input**: Model detail page
**Output**: Year + VIN restriction combinations

Parses tables with year ranges and optional VIN restrictions.

## Level 4: CategoriesParser (`crawler/parsers/categories.py`)

**Input**: Year/variant page
**Output**: 10 parts categories (Accessors, Engine, Fuel, etc.)

Each category has: name, category_index (0-9), catalog_url

## Level 5: SubgroupsParser (`crawler/parsers/subgroups.py`)

**Input**: Category page
**Output**: Subgroup entries (main_group, illustration_number, description)

Parses the subgroup table with columns: MG, Ill-No, Description, Remark, Model Data

## Level 6: PartsParser (`crawler/parsers/parts.py`)

**Input**: Subgroup detail page
**Output**: Individual parts

Parses the parts table with columns: Position, Part Number, Description, Remark, Quantity, Model Data

## Level 2 Auto-Detection (engine.py)

The `CrawlEngine._pick_level2_parser()` method auto-detects page format:

| h1 Heading | Parser | Action |
|------------|--------|--------|
| "Market" / "Brand" / "Assortment class" | None | Gateway - follow links, re-enqueue at L2 |
| "Catalog" | BrandModelsParser | Table-based (VW-style) |
| "Model" + table rows | BrandModelsParser | Table-based after market selection |
| "Model" + links | LinkModelsParser | Link-based (BMW, Ford) |
| "Year and region selection" / "Year / Model" | None | Year-first format - skipped |

## Known Parser Issues

1. **Long category names**: Some Ford categories have names > 100 chars. Fixed by using VARCHAR(300).
2. **Long production_date**: Some Mercedes models have production_date > 50 chars. Fixed by using VARCHAR(200).
3. **Year-first brands**: Some brands (rare) show years before models. These are currently skipped.
